# GitHub Authentication Setup

This configuration uses GitHub CLI authentication instead of SSH keys for git operations.

## Initial Setup

1. Authenticate with GitHub CLI:
```bash
gh-auth-refresh
```

This command will:
- Logout from any existing GitHub CLI session
- Login to GitHub CLI (opens browser for authentication)
- Setup git credential helper in `~/.private/git/credential/config`

## How It Works

- **No SSH keys required**: Git operations use HTTPS protocol with `gh auth` credential helper
- **Automatic URL conversion**: SSH URLs (`git@github.com:`) are automatically converted to HTTPS (`https://github.com/`)
- **Credential storage**: GitHub credentials are stored via `gh auth` and used for all git operations
- **Works for private repositories**: Full access to private repositories with proper token scopes (repo, read:org, gist)

## Configuration Files

- `shared-home-manager/taco/git/default.nix`: Configures automatic SSH to HTTPS URL conversion
- `linux/home-manager/fish/functions.nix`: Defines `gh-auth-setup` and `gh-auth-refresh` functions
- `~/.private/git/credential/config`: Writable git config for credential helpers (auto-generated by `gh auth setup-git`)

## For Golang Private Repositories

Set the `GOPRIVATE` environment variable for your private Go modules:
```bash
export GOPRIVATE=github.com/your-org/*
```

Go will automatically use the git credential helper when fetching private modules.
