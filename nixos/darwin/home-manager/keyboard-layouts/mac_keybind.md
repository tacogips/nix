# Keybind

US keyboard layout.

Each row shows:
1. English
2. English + Shift
3. Japanese
4. Japanese + Shift

`2` and `4` are the Shift rows.
`む` cannot be typed on this layout because there is no dedicated key for it on a US keyboard.

## Notes

- Mac and Linux are based on the same physical US keyboard.
- Linux entries below ignore `<esc>` and `<blank>` because those keys are not part of the comparable typing area in the Mac table.
- ASCII mapping should be identical on both.
- Differences are in Japanese kana assignment, especially `む`, punctuation, and some shifted kana.

## Correspondence

- ASCII key positions correspond exactly.
- Linux has a direct `む` on Row 1 right end, but Mac US kana input does not.
- Linux Row 1 right starts with `へ`, while Mac Row 1 right starts with `゜`.
- Linux Row 4 left starts with `つ`, while Mac Row 4 left starts with `っ`.
